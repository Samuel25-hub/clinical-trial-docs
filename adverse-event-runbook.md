# Adverse Event Reporting Runbook

## Purpose

This runbook provides guidance for identifying, documenting, classifying, and reporting adverse events during the HX-214 Phase II clinical trial.

The goal of this process is to support participant safety and regulatory compliance.

---

## Definitions

### Adverse Event (AE)

Any unfavorable medical occurrence experienced by a study participant during the trial, whether or not it is related to study treatment.

### Serious Adverse Event (SAE)

An adverse event that:
- results in hospitalization
- causes significant disability
- is life-threatening
- results in death
- requires medical intervention to prevent serious harm

---

## Reporting Workflow

### Step 1: Identify the Event

Study personnel should:
- document participant symptoms
- gather clinical details
- determine event timing
- review treatment relationship

---

### Step 2: Assess Severity

| Severity Level | Description |
|---|---|
| Mild | Temporary symptoms with minimal impact |
| Moderate | Symptoms affecting daily activities |
| Severe | Significant medical risk or intervention required |

---

### Step 3: Determine Reporting Requirements

### Mild Events
- document in study records
- monitor participant status
- report during routine review

### Moderate Events
- notify investigator
- document event details
- evaluate need for follow-up care

### Serious Adverse Events (SAEs)
- notify investigator immediately
- initiate sponsor reporting procedures
- complete SAE documentation
- follow regulatory reporting timelines

---

## SAE Escalation Timeline

| Event Type | Reporting Timeline |
|---|---|
| Serious Adverse Event | Within 24 hours |
| Life-Threatening Event | Immediate notification |
| Participant Death | Immediate notification |

---

## Required Documentation

The following information should be recorded:
- participant identifier
- event description
- severity classification
- onset date
- outcome status
- treatment relationship
- corrective actions taken

---

## Decision Guidance

### Was the participant harmed?

- If NO → document and continue monitoring
- If YES → evaluate severity level

### Is hospitalization required?

- If YES → classify as Serious Adverse Event (SAE)

### Is immediate intervention required?

- If YES → notify investigator immediately

---

## Confidentiality and Privacy

All adverse event documentation must:
- protect participant confidentiality
- avoid unauthorized disclosure
- follow institutional privacy procedures
- comply with study reporting requirements

---

## Regulatory References

This workflow references concepts modeled on:
- ICH E6 Good Clinical Practice (GCP)
- FDA 21 CFR Part 312
- clinical trial safety reporting workflows
